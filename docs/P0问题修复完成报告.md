#### 修复后（统一为5步法）
```
## 📊 分析框架（5步法）

### 第1步：深度分析复盘分析师的策略排名（🔴 最重要！必须第一步执行）
- 这是最重要的决策依据，必须优先分析
- 查看复盘分析师提供的策略胜率排名
- 识别优秀策略（胜率>60%）和不佳策略（胜率<50%）
- 分析成功案例和失败案例
- 理解策略的适用场景
- 如果历史交易记录<10笔，进入冷启动模式

### 第2步：计算推荐数量
- 工具：query_total_assets + calculate_recommended_count
- 推荐数量规则：<10万推荐1只，10-30万推荐2只，>30万推荐3只
- 即使满仓，也要推荐股票（投资决策官会对比持仓和推荐）

### 第3步：分析市场环境（市场阶段识别 + 市场情绪与热点分析）
- 3.1 市场阶段识别（如果有identify_market_phase工具）
- 3.2 涨停板分析（如果有"分析昨日涨停板"工具）
  * 📋 在分析输出中完整展示涨停板分析结果
  * 如果API返回错误（如周末无数据），记录错误信息并继续
  * 如果没有此工具，跳过此步骤
- 3.3 获取市场情绪数据（使用get_market_sentiment工具）
- 3.4 搜索最新市场新闻（使用search_market_news工具）

### 第4步：综合判断最优策略
- 综合复盘分析师的策略排名、市场情绪、新闻热点
- 优先考虑历史胜率高的策略
- 根据当前市场环境调整策略
- 不要使用固定规则，要根据历史数据和当前市场灵活决策

### 第5步：动态生成具体的筛选参数
- 根据选定的策略，生成筛选参数
- 参数必须是具体数值（如涨幅2-9%，换手率>3%）
- ✅ 重要：如果策略涉及特定板块或概念，必须填写sectors或concepts参数
- 必须调用3次"保存Agent上下文"工具（推荐数量、策略名称、筛选参数）
```

#### 修复统计
