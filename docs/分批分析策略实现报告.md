# 分批分析策略实现报告

## 📋 问题背景

**用户反馈**：
> "Context length exceeded. Summarizing content to fit the model context window."

**问题分析**：
- ✅ 智能选股师筛选出30-50只候选股
- ✅ 多维分析师分析所有候选股（6维分析）
- ❌ **总上下文长度超过DeepSeek的64K限制**
- ❌ **CrewAI自动压缩上下文，丢失大量信息**

---

## 💡 解决方案：分批分析 + 紧凑模式

### 核心策略

**分批分析**：
- 每批最多10只股票
- 逐批调用`analyze_stocks_parallel`工具
- 所有批次分析完后，汇总结果

**紧凑模式**：
- `compact_mode=True`（默认）
- 每只股票输出减少80%（2000字符 → 400字符）
- 只保留关键信息

```python
@tool("并行分析多只股票")
def analyze_stocks_parallel(stock_codes: str, compact_mode: bool = True) -> str:
    """
    并行分析多只股票的技术、资金、基本面、新闻、逐笔交易、社区情绪六个维度

    Args:
        stock_codes: 股票代码列表，逗号分隔，如'600000,000001,002163'
        compact_mode: 是否使用紧凑模式（默认True，减少输出长度，避免上下文超长）

    Returns:
        所有股票的综合分析结果
    """
```

**紧凑模式输出格式**：

## 🎯 最终效果

**预期流程**：
```
智能选股师 → 筛选30-50只候选股 → 保存到AgentContext
    ↓
多维分析师 → 读取AgentContext → 分批分析
    ↓
第1批：分析前10只（8K上下文）
第2批：分析第11-20只（8K上下文）
第3批：分析第21-30只（8K上下文）
第4批：分析第31-40只（8K上下文）
第5批：分析第41-50只（8K上下文）
    ↓
汇总所有结果 → 传递给风险管理官
    ↓
风险管理官 → 评估所有30-50只 → 批准/拒绝
    ↓
投资决策官 → 从批准的股票中选择Top N
```

**关键优势**：
- ✅ 可以分析50+只候选股
- ✅ 完全避免上下文压缩
- ✅ 不丢失任何信息
- ✅ 保留关键信息（紧凑模式）
- ✅ 提升分析质量（更多候选股）
