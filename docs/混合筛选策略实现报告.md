# 混合筛选策略实现报告

## 📋 问题背景

用户反馈了两个关键问题：

### 问题1：候选股数量不一致
- **文档说明**：`smart_recommendation_crew.py` 第500行设置 `max_results: 50`
- **实际代码**：`smart_agents.py` 多处使用15-20只的标准
- **影响**：AI筛选出50只股票，但评估标准却是15-20只，导致逻辑混乱

### 问题2：板块筛选逻辑不合理
- **当前逻辑**：
  - 只处理最多2个板块（`sector_names[:2]`）
  - 每个板块只返回3只股票（`top_n=3`）
  - 如果板块筛选不到足够股票，直接跳过该板块
- **用户质疑**："这些板块如果拿不到足够的股票，你就把板块去掉了？这样真的好吗？"
- **问题**：
  - 最多只能得到6只候选股（2个板块 × 3只）
  - 远远不够50只的目标
  - 放弃板块会丢失热点信息

## ✅ 解决方案：混合筛选策略

### 策略设计

**核心思想**：板块筛选（优先） + 全市场筛选（补充）

```
┌─────────────────────────────────────────────────────────┐
│                    混合筛选策略                          │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  第1步：板块筛选（从热点板块筛选高质量候选股）           │
│  ├─ 目标：15-25只                                        │
│  ├─ 方法：dynamic_screen_stocks(sectors="AI,新能源")     │
│  └─ 优势：聚焦热点，质量高                               │
│                                                          │
│  第2步：全市场筛选（补充到30-50只）                      │
│  ├─ 条件：如果板块筛选 < 30只                            │
│  ├─ 方法：dynamic_screen_stocks(sectors="")              │
│  └─ 优势：确保数量，分散风险                             │
│                                                          │
│  第3步：合并去重                                         │
│  ├─ 按股票代码去重                                       │
│  ├─ 按策略要求排序                                       │
│  └─ 最终：30-50只候选股                                  │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 实现细节

#### 1. 统一候选股数量标准（50只）

**修改文件**：`src/agents/smart_agents.py`

**修改内容**：
- 第1791-1806行：`max_results` 从20改为50
- 第1816-1820行：返回数量从"15-20只"改为"30-50只"
- 第1831-1837行：候选股数量评估标准更新
  - 优秀：≥30只（原来≥15只）
  - 良好：20-29只（原来10-14只）
  - 一般：10-19只（原来5-9只）
  - 不合格：<10只（原来<5只）

#### 2. 更新筛选逻辑为混合策略

**修改文件**：`src/agents/smart_agents.py`

**新增内容**（第1810-1867行）：
```
## 📋 你的筛选逻辑（混合策略：板块筛选 + 全市场筛选）

第1步：从市场情报官的输出中提取筛选参数
第2步：板块筛选（如果市场情报官提供了板块/概念）
第3步：全市场筛选（补充候选股到30-50只）
第4步：合并去重
第5步：评估筛选结果质量
```

#### 3. 修复板块筛选工具逻辑

**修改文件**：`src/agents/tools/pre_market_news_tools.py`

**修改内容**（第280-291行）：
- **原逻辑**：如果筛选不到股票，返回错误
- **新逻辑**：如果筛选不到股票，返回空列表（不报错）
- **改进**：允许返回少于 `top_n` 的股票（如果候选股不足）

```python
# 修改前
if not candidate_stocks:
    return f"❌ 板块 '{sector_name}' 中未找到符合条件的股票{filter_msg}"

# 修改后
if not candidate_stocks:
    logger.warning(f"⚠️ 板块 '{sector_name}' 未找到符合条件的股票")
    return f"⚠️ 板块 '{sector_name}' 中未找到符合条件的股票{filter_msg}\n📋 候选股代码："
```

## 📊 效果对比

### 修改前
```
板块筛选：最多2个板块 × 3只 = 6只
全市场筛选：无
最终结果：6只（远低于50只目标）
问题：候选股太少，选择空间小
```

### 修改后
```
板块筛选：3-5个板块 × 5-10只 = 15-50只
全市场筛选：补充到30-50只
最终结果：30-50只（符合目标）
优势：候选股充足，既有热点又有分散
```

## 📝 总结

### 核心改进
1. ✅ 统一候选股数量标准为50只
2. ✅ 实现混合筛选策略（板块 + 全市场）
3. ✅ 修复板块筛选工具逻辑（不再因筛选不到股票而报错）
4. ✅ 更新质量评估标准和示例

### 优势
- **数量保障**：确保获得30-50只候选股
- **质量优先**：优先从热点板块筛选
- **风险分散**：全市场补充，避免过度集中
- **容错性强**：即使板块筛选失败，全市场筛选也能兜底
