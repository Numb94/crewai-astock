## 🔴 问题描述

### 用户反馈的问题

**场景**：
1. 昨天推荐股票A（买入）
2. 今天卖了股票A（可能是止损，K线冲高回落）
3. 今天又继续推荐股票A
4. **问题**：为什么会重复推荐刚卖出的股票？

### 问题本质

**现有逻辑的缺陷**：
```python
# Task 6（投资决策官）第3步
3. 🔴 查询当前持仓，过滤已持仓股票
   - 使用工具：query_current_positions
   - 只过滤了 status='holding' 的股票
   - ❌ 但没有过滤 status='sold' 的股票！
```

**问题流程**：
1. 昨天推荐股票A → 买入（status='holding'）
2. 今天卖出股票A → status变为'sold'
3. 今天再次推荐时，只过滤`status='holding'`的股票
4. 股票A的status是'sold'，**不会被过滤**
5. **结果**：再次推荐股票A！（即使K线冲高回落）

### 更严重的问题

1. **没有考虑卖出原因**：
   - 止损卖出：说明股票走势不好，不应该短期内再推荐
   - 止盈卖出：可能还有机会，但需要冷却期
   - 手动卖出：用户主动卖出，可能有特殊原因

2. **没有持仓对比分析**：
   - 没有对比"持仓股票 vs 推荐股票"的质量
   - 没有给出"继续持有"或"换股"的建议

## ✅ 解决方案

### 1. 创建新工具：`query_recently_sold_positions`

**文件**: `src/agents/tools/position_tools.py`

**功能**：
- 查询最近N天内卖出的股票
- 根据卖出原因设置不同的冷却期
- 返回冷却期内的股票代码列表

**冷却期规则**：
```python
cooldown_rules = {
    '止损': 7,  # 止损卖出：7天冷却期（股票走势不好）
    '止盈': 3,  # 止盈卖出：3天冷却期（可能还有机会）
    '手动': 5,  # 手动卖出：5天冷却期（用户主动卖出）
    '其他': 5   # 其他原因：5天冷却期
}
```

**返回格式**：
```json
{
  "total": 2,
  "in_cooldown_count": 1,
  "in_cooldown_codes": ["600000"],
  "stocks": [
    {
      "stock_code": "600000",
      "stock_name": "浦发银行",
      "sell_date": "2025-11-19",
      "sell_price": 10.50,
      "sell_reason": "止损",
      "buy_price": 11.50,
      "profit_loss_pct": -8.70,
      "cooldown_days": 7,
      "days_since_sell": 1,
      "remaining_cooldown": 6,
      "in_cooldown": true
    }
  ],
  "message": "最近7天内卖出2只股票，其中1只仍在冷却期内"
}
```
